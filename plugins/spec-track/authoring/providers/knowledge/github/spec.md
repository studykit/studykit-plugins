# GitHub Knowledge Spec Authoring

Provider-specific binding for `spec` knowledge pages stored as repository Markdown files.

Read after:

- `../../../contracts/knowledge/spec.md`
- `./convention.md`

## Scope

Use this binding for GitHub repository Markdown `spec` pages. The final body structure is listed below.

## Final body structure

Use this final GitHub repository Markdown body structure for `spec` pages.

Common required sections are defined by `../../../contracts/knowledge/spec.md`:

- `## Context`
- `## Specification`

Common optional sections are defined by `../../../contracts/knowledge/spec.md` and `../../../contracts/knowledge/body.md`:

- `## Supersedes`
- `## Related Work`
- `## Open Questions`
- `## Consequences`
- `## Examples`
- `## Decision Log`
- `## Rejected Alternatives`
- `## Change Log`
- `## Sources`
