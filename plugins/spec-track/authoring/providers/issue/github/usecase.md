# GitHub Issue Use Case Issue Authoring

Provider-specific binding for `usecase` issues stored as GitHub Issues.

Read after:

- `../../../contracts/issue/usecase.md`
- `./convention.md`
- `./relationships.md`

## Scope

Use this binding for the GitHub issue side of `usecase`. The final body structure is listed below.

## Final body structure

Use this final GitHub body structure for `usecase` issues.

Common required sections are defined by `../../../contracts/issue/usecase.md`:

- `## Description`
- `## Actors`
- `## Current Draft`
- `## Open Questions`

Common optional sections are defined by `../../../contracts/issue/usecase.md` and `../../../contracts/issue/body.md`:

- `## Resume`

GitHub label is automatically set to `usecase`.
