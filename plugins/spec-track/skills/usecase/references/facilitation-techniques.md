# Facilitation Techniques

The discovery loop (`./discovery-loop.md`) drives the default
question rhythm — pick the least-clear gap, ask one focused
question, refine. When that rhythm starts circling — the user
repeats answers, every question lands in the same place, the scope
keeps growing without a center — shift the interview mode for a
turn or two to break the pattern. Return to the default rhythm once
the conversation moves again.

## Mode shifts

### Contrarian

Challenge an unexamined assumption that is shaping every answer.

Example prompts:

- "You keep assuming X. What if X isn't true?"
- "What if the opposite were true?"
- "Who would build this without that constraint, and what would
  change?"

Use when a strong assumption (a chosen tool, a fixed actor, an
existing flow) is implicit in every answer and has not been
questioned.

### Simplifier

Strip away complexity to find the core.

Example prompts:

- "If you could only solve one thing here, what would it be?"
- "What's the smallest version of this that still matters to the
  actor?"
- "Which step would still hurt if everything else worked?"

Use when scope keeps growing without a clear center, or when the
user is enumerating features instead of describing actor outcomes.

### Reframer

Look at the use case from a different angle.

Example prompts:

- "Who would hate this, and why?"
- "If this succeeds wildly, what new problem does it create for the
  actor?"
- "What does this look like from the perspective of the actor who
  doesn't benefit?"

Use when the conversation has gone deep in one direction without
exploring alternatives, or when every use case so far covers the
same actor's happy path.

## Discipline

- Shift mode for a turn or two, not for the whole interview. The
  default discovery loop is the rhythm — the modes are interrupts.
- Name the shift to the user when it might surprise them ("Let me
  push back on this for a moment …"). Use cases discovered through a
  shift are no less valid; the user just needs to know the question
  is intentional.
- Do not stack modes. Pick one, run it, then return to the discovery
  loop. Stacking ("contrarian, then reframer, then simplifier") is
  the conversational equivalent of asking three questions at once.
