# Verification Checklist

Run each check and record outcomes locally for the issue-handling step. No wiki page is written.

- **Build** — build command exits 0.
- **Run** — launch command starts the app without error; clean up.
- **Dev loop** — trivial source edit → rebuild succeeds and the app re-launches.

Test execution is out of scope here. `/a4:ci-setup` owns runner installation, fixtures, and the test-execution checklist captured in `a4/ci.md`.
