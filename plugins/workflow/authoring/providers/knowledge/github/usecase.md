# GitHub Knowledge Use Case Page Authoring

Provider-specific binding for `usecase` knowledge pages stored as repository Markdown files.

Read after:

- `../../../contracts/knowledge/usecase.md`
- `./convention.md`

## Scope

Use this binding for GitHub repository Markdown `usecase` pages. The final body structure is listed below.

## Final body structure

Use this final GitHub repository Markdown body structure for `usecase` pages.

Common required sections are defined by `../../../contracts/knowledge/usecase.md`:

- `## Goal`
- `## Actors`
- `## Situation`
- `## Flow`
- `## Expected Outcome`

Common optional sections are defined by `../../../contracts/knowledge/usecase.md` and `../../../contracts/knowledge/body.md`:

- `## Validation`
- `## Error Handling`
- `## Related Work`
- `## Supersedes`
- `## Change Log`

Link the source issue and implementation work with Markdown links or full GitHub issue URLs.
