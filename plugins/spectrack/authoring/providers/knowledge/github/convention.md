# GitHub Knowledge Page Conventions

Provider-specific rules for knowledge pages stored as repository Markdown files. Read after the type's `../../../contracts/knowledge/` contract and `../../../contracts/knowledge/body.md`, which define the sections and the `Change Log` rule.

- Keep section anchors stable — repository links may target headings.
- Link work items (issues, reviews, tasks, use case pages) in Markdown link form or as full GitHub issue URLs.
- Render list-style content — for example a decision index's `Decisions` entries — as GFM list items.
