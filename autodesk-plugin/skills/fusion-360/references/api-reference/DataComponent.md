## Description

An object that provides ID's that can be used to access the component using MFG DM API. These ID's are generated by the MFG DM API when a file is saved. For that reason, they are not available from a component before its parent file is saved.

## Methods

|  |  |
| --- | --- |
| Name | Description |
| [classType](DataComponent_classType.htm) | Static function that all classes support that returns the type of the class as a string. The returned string matches the string returned by the objectType property. For example if you have a reference to an object and you want to check if it's a SketchLine you can use myObject.objectType == fusion.SketchLine.classType(). |

## Properties

|  |  |
| --- | --- |
| Name | Description |
| [component](DataComponent_component.htm) | Get the fusion.Component that corresponds to this data component. |
| [id](DataComponent_id.htm) | Gets the unique ID of this DataComponent. This ID can be used within the MFG DM API. This ID is created by the MFG DM API when the component is first saved. |
| [isValid](DataComponent_isValid.htm) | Indicates if this object is still valid, i.e. hasn't been deleted or some other action done to invalidate the reference. |
| [mfgdmModelId](DataComponent_mfgdmModelId.htm) | Get the MFGDM model Id. |
| [objectType](DataComponent_objectType.htm) | This property is supported by all objects in the API and returns a string that contains the full name (namespace::objecttype) describing the type of the object. |
| [timestamp](DataComponent_timestamp.htm) | Get the resolution timestamp. |
| [versionId](DataComponent_versionId.htm) | Returns the unique ID of this file version of the DataComponent. This ID is updated whenever the document the component exists within is saved. |
